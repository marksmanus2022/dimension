Step 1 — Cabinet Configuration

Enter the number of cabinets you want to deploy, then specify the servers for each cabinet.

Allocation Principles
  1. Kickstart — Always compute 0 in Cabinet 1. Auto-designated on layout generation. Kickstart computes cannot host VMs.
  2. RTE — Auto-assigned on layout generation: 5th compute in Cabinet 1, 5th compute in Cabinet 2. RTE computes cannot host VMs.
  3. Controller — Auto-assigned on layout generation: 4th compute in Cabinet 1; 3rd and 4th computes in Cabinet 2. Controller computes cannot host VMs.
  4. OAM — Auto-assigned on layout generation. The highest-numbered Gold CPU compute in each of Cabinets 4, 5 and 6 is designated OAM. OAM computes cannot host VMs.
  5. WMG — Auto-assigned (when enabled) before SRIOV. Exactly 9 computes, must be Gold CPU. Excludes Kickstart, RTE, Controller, OAM. Within each cabinet, picks alternate between the lowest and highest available computes for maximum spread; round-robin across cabinets to maximise cabinet distribution.
  6. SRIOV — Auto-assigned after WMG. Cabinet-pair quotas: C1+C2 ≤ 12, C3+C4 ≤ 11, C5+C6 ≤ 8 (total cap 31). Excludes Kickstart, RTE, Controller, OAM, and WMG. Within each cabinet, picks alternate between the lowest and highest available computes to maximise physical spread.
  7. Gold CPU — Manually specified. Must be more than 3 computes. Colours the server number magenta in the layout; does not change the slot background colour.
